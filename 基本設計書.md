# iPhoneからWindows PCへの動画転送アプリ 基本設計書

**バージョン**: 1.0
**作成日**: 2026-03-08
**プロジェクト名**: MovieSend

---

## 1. プロジェクト概要

### 1.1 目的

iPhoneのPhotosライブラリに保存された動画を、同一LAN環境にあるWindows PCへ簡単・高速・安全に転送するiOSアプリを開発する。ケーブル不要、追加ソフトウェアのインストール不要で、PINコードを入力するだけで転送を開始できる体験を目指す。カメラ不要のため、カメラを持たないPCでも利用できる。

### 1.2 スコープ

**対象**
- iPhone（iOS 16.0以上）
- Windows PC（Chrome / Edge ブラウザ）
- 同一Wi-Fiネットワーク内での転送

**機能スコープ**
- Photosライブラリからの動画一覧表示・選択
- PIN認証による接続確立（mDNS自動検出 + 4桁英数字PINコード）
- HTTP経由での動画ダウンロード（Rangeリクエスト対応）
- 転送進捗表示
- セッション管理・認証

**対象外（非スコープ）**
- 写真（静止画）の転送
- クラウド経由の転送
- Bluetooth転送
- Mac / Android 対応
- バックグラウンドでの継続転送

### 1.3 非機能要件

| 要件 | 目標値 |
|------|--------|
| 転送速度 | Wi-Fi 5GHz 環境で 50 Mbps 以上 |
| 対応動画サイズ | 最大 10 GB / ファイル |
| 接続確立時間 | PIN入力から30秒以内 |
| セキュリティ | LAN内限定、ワンタイムトークン認証 |
| iOS最低バージョン | iOS 16.0 |
| 同時転送 | 1ファイルずつ順次転送 |
| アプリサイズ | 30 MB 以下 |

## 2. 技術スタック選定

### 2.1 iPhone側フレームワーク比較

| 観点 | Swift / SwiftUI | React Native | Flutter |
|------|-----------------|--------------|---------|
| PhotosKit 統合 | ◎ ネイティブ直接利用 | △ サードパーティ経由 | △ サードパーティ経由 |
| Network.framework (HTTPサーバー) | ◎ 直接利用可能 | ✕ 利用不可 | ✕ 利用不可 |
| バックグラウンド制約への対応 | ◎ 公式APIで最適化 | △ 間接的 | △ 間接的 |
| パフォーマンス | ◎ | △ JSブリッジあり | ○ |
| ファイルサイズ | ◎ 小さい | △ ランタイム含む | △ ランタイム含む |
| 開発コスト | ○ | ◎ Web知識流用 | ○ |

**推奨: Swift / SwiftUI**

PhotosKit・Network.framework・AVFoundation など iOS ネイティブ API を直接利用できる。HTTPサーバー実装に必要な低レベルネットワーク処理も Swift で完結し、クロスプラットフォームフレームワークでは実現困難。

### 2.2 PC側クライアント比較

| 観点 | ブラウザ (HTML/JS) | Electron | Python スクリプト | .NET アプリ |
|------|-------------------|----------|------------------|-------------|
| インストール不要 | ◎ | ✕ 要インストール | △ Python要インストール | ✕ 要インストール |
| 開発コスト | ◎ | ○ | ○ | △ |
| UI品質 | ◎ CSS/JSで自由 | ◎ | △ CLI中心 | ○ |
| 配布容易性 | ◎ URLのみ | △ 配布物大 | △ | △ |
| セキュリティ | ○ CORS制御 | ○ | ○ | ○ |

**推奨: ブラウザ (HTML / CSS / Vanilla JS)**

PC側に追加ソフトウェアのインストールを一切不要にできる。iPhone側のHTTPサーバーが静的HTML/JS/CSSも配信するため、ユーザーはブラウザでURLを開きPINを入力するだけで使用開始できる。

### 2.3 転送プロトコル比較

| 観点 | HTTP/1.1 | WebSocket | FTP |
|------|----------|-----------|-----|
| ブラウザ標準対応 | ◎ | ◎ | ✕ |
| Rangeリクエスト（レジューム） | ◎ | ✕ 自前実装 | ◎ |
| 実装複雑度 | ◎ シンプル | ○ | △ 2ポート |
| ファイアウォール通過 | ◎ 80/8080 | ◎ | △ ポート問題 |
| 進捗表示 | ◎ Content-Length + fetch | ○ | △ |

**推奨: HTTP/1.1（Rangeリクエスト対応）**

標準的なHTTPでブラウザとの親和性が最高。`Content-Range` ヘッダーによるレジューム転送が実装でき、大容量動画の転送中断にも対応できる。

### 2.4 デバイス探索方式比較

| 観点 | mDNS / Bonjour | 手動IP入力 | QRコード |
|------|----------------|-----------|----------|
| ユーザー体験 | ◎ URL自動解決 | △ IP入力が必要 | ✕ PCカメラが必要 |
| カメラ不要 | ◎ | ◎ | ✕ PCカメラ必須 |
| 認証との統合 | ○ PINをブラウザで入力 | ○ PINをブラウザで入力 | △ |
| 信頼性 | △ ルーター設定依存 | ◎ | ◎ |
| 実装コスト | ○ Network.framework | ◎ | ◎ CoreImage |

**推奨: mDNS（プライマリ）+ 短縮PINコード認証（セカンダリ）**

iPhoneがHTTPサーバーを起動し、4桁英数字PINをアプリ画面に大きく表示する。PCユーザーはブラウザで `http://moviesend.local:8080/` を開き（mDNS自動解決）、表示されたPINを入力するだけで認証済み接続を確立。mDNSが利用できない環境では、iPhone画面に表示されるIPアドレスを手動入力してからPINを入力する。PCカメラは一切不要。

### 3.2 エンドポイント設計

| メソッド | パス | 説明 | 認証 |
|----------|------|------|------|
| GET | `/` | Web UI (HTML) を返す | トークン |
| GET | `/api/videos` | 動画一覧 JSON | トークン |
| GET | `/api/videos/{id}/thumbnail` | サムネイル画像 | トークン |
| GET | `/api/videos/{id}/download` | 動画ファイル本体 (Range対応) | トークン |
| GET | `/api/status` | サーバー状態確認 | トークン |

#### 認証方式

PIN入力成功後、レスポンスCookieに `session=<SESSION_TOKEN>` をセットする。
以降のリクエストはCookieで自動送信される。クエリパラメータ `?token=<SESSION_TOKEN>` またはヘッダー `X-Session-Token: <SESSION_TOKEN>` でも受け付ける。

PIN認証専用エンドポイントを追加:

| メソッド | パス | 説明 | 認証 |
|----------|------|------|------|
| POST | `/api/auth` | PINを検証しCookieセッションを開始 | 不要 |

### 3.4 HTTPサーバー実装方針

- `Network.framework` の `NWListener` でTCPソケットをリッスン
- HTTP/1.1 パーサーを自前実装（シンプルなGETのみ対応）
- `Range: bytes=N-M` ヘッダーを解析し、PhotosKit から部分読み出しして返す
- コネクション数は最大3に制限（過負荷防止）

## 4. セキュリティ設計

### 4.1 セッショントークン

- 起動ごとに `UUID().uuidString` で128ビットランダムセッショントークンを生成
- 別途、4桁英数字PIN（例: `A3F7`）を生成しiPhone画面に大きく表示する
- PCユーザーがPINを入力すると、サーバーがPINとセッショントークンを照合し、レスポンスのCookieにセッショントークン（`session=<UUID>`）をセットする
- 以降のリクエストはCookieのセッショントークンで認証する（クエリパラメータでも可）
- トークンは定数時間比較（タイミング攻撃対策）で検証
- 無効なトークンには `403 Forbidden` を返す

### 4.2 セッション有効期限

- HTTPサーバー起動から **30分** でセッション自動失効
- アプリがバックグラウンドに移行した時点でサーバーを停止
- スクリーンがオフになった場合もサーバーを停止

### 4.3 LAN限定アクセス

- `NWListener` はLANインターフェース（Wi-Fi）のみバインド
- ループバック（127.0.0.1）のみからのアクセスは拒否
- インターネットへのルーティングは行わない（iPhoneがサーバーのため、ルーターでのNATは関係しない）

### 4.4 その他

- HTTPS は実装しない（自己署名証明書はブラウザ警告が出るため）。代わりにトークン認証でセキュリティを確保
- アクセスログはオンメモリのみ（永続化しない）
- PhotosKit のアクセス権限は動画のみ要求

## 8. 技術的注意事項

### 8.1 iOSバックグラウンド制約

iOSはバックグラウンドでのネットワーク処理を厳しく制限している。

- アプリがバックグラウンドに移行すると **30秒以内** にネットワーク処理が停止される
- `UIApplication.shared.beginBackgroundTask` を使っても最大 **30秒** の猶予しか得られない
- **対策**: UIに「転送中はアプリを前面に表示してください」と明示。バックグラウンド移行を `ScenePhase` で検知しサーバーを即座に停止する

```swift
// バックグラウンド移行の検知例
.onChange(of: scenePhase) { phase in
    if phase != .active {
        httpServer.stop()
    }
}
```

### 8.2 HEVC（H.265）互換性

iPhoneで撮影した動画は HEVC (H.265) コーデックが使われるが、Windows での再生には制限がある。

- Windows 10/11 の Edge/Chrome ブラウザは HEVC をネイティブ再生 **できない**
- ダウンロードしたファイルは **VLC** などのプレーヤーで再生可能
- App内でのトランスコード（HEVC→H.264）は処理時間・バッテリー消費が大きいため **非推奨**
- **対策**: Web UI に「VLC推奨」の案内を表示。必要に応じてAVAssetExportSessionでの変換オプションをフェーズ4以降で検討

### 8.3 PhotosKit 使用上の注意

```swift
let options = PHVideoRequestOptions()
options.isNetworkAccessAllowed = false  // iCloud動画はダウンロードしない

    // AVURLAsset のURLを直接HTTP Rangeリクエストで読み出す
}
```

### 8.4 大容量ファイルのメモリ管理

10GBの動画ファイルを一括でメモリに読み込むことはできない。

- HTTPレスポンスはチャンク転送で実装（`Transfer-Encoding: chunked` または固定バッファ）
- バッファサイズは **64KB〜256KB** 程度に設定
- `AVAssetReader` + `AVAssetReaderTrackOutput` でフレームバイフレームの読み出しは避ける（オーバーヘッドが大きい）
- `FileHandle` や `InputStream` で直接ファイルを分割読み出しする方が高速

### 8.5 App Store 審査

- **NSPhotoLibraryUsageDescription**: Info.plistに使用目的を明記（「動画をPCに転送するため」）
- **Privacy Manifest (PrivacyInfo.xcprivacy)**: iOS 17以降で必須。`NSPrivacyAccessedAPITypes` に PhotosKit を記載
- **ネットワークサーバー機能**: App Store ガイドライン上の制限はないが、審査説明文に「同一LAN内でのP2P転送のみ」と記載することを推奨
- **最小化の原則**: Photos権限は動画のみ要求（`PHAssetMediaTypeVideo` フィルタ）

### 8.6 IPアドレス取得

```swift
// Wi-FiインターフェースのIPアドレスを取得する例
import Network

func getLocalIPAddress() -> String? {
    var address: String?
    var ifaddr: UnsafeMutablePointer<ifaddrs>?
    guard getifaddrs(&ifaddr) == 0 else { return nil }
    defer { freeifaddrs(ifaddr) }

    var ptr = ifaddr
    while ptr != nil {
        let interface = ptr!.pointee
        let addrFamily = interface.ifa_addr.pointee.sa_family
        if addrFamily == UInt8(AF_INET),
           let name = String(validatingUTF8: interface.ifa_name),
           name == "en0" {  // Wi-Fiインターフェース
            var addr = interface.ifa_addr.pointee
            var hostname = [CChar](repeating: 0, count: Int(NI_MAXHOST))
            getnameinfo(&addr, socklen_t(interface.ifa_addr.pointee.sa_len),
                        &hostname, socklen_t(hostname.count), nil, 0, NI_NUMERICHOST)
            address = String(cString: hostname)
        }
        ptr = interface.ifa_next
    }
    return address
}
```
